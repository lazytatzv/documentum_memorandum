# Linux

## Prerequisites

OS(Operating System)の構成要素、Kernelの役割について軽く把握しておくこと.

一応簡単に書いておくが、

OSとはコンピュータ全体を管理するソフトウェアであり、その中核がKernelである. Kernel + ツール群(コマンドとか) + ライブラリ = OSみたいな感じ.
Linux界隈ではOSの中のKernel以外の要素をよくUserLandと呼ぶ.

Kernelの基本要素としては、メモリ管理、プロセス管理、ファイルシステム、デバイス制御を覚えておけば一旦はよい. 
最初にここを深く掘ると抜け出せなくなるので徐々に把握していければ問題なし.

## 心構え

多少長いが、大切なので先に書いておく.

1. 理解できなくて当たり前

最初からすべて理解しようとするのは無理. というか、3000万行以上もあるLinux Kernelの全容を把握しようとすること自体が無謀.

2. 全部を一気に勉強しようとしない

ps, mm, fs, dev全部一気にやろうとすると頭がパンクする. 点->線->面で理解を深めていく. あといきなり抽象的なところからいかない.

3. とりあえず触る

Linuxには様々なコマンドがあるが、とりあえず触ってみること. 普通にやってれば壊れることはない. いや、何なら壊してもいい.

4. 抽象と具体を往復する

プロセスが何か勉強するだけでは使えないし、`ps`をテキトーに使っているだけでは理解できたと言えない. 多少勉強しつつ、コマンド打ちつつという塩梅が大切.

5. 哲学を理解する

Linux、というかUnixの哲学を中心に据える. いかに優れており汎用的な考え方かわかるはず.

## 目標

1. OSの仕組みをざっくり理解する

2. コマンドを使ってデバッグできる

3. Unix哲学を理解する

## どうやってLinuxに慣れていくのか

正直なところ実機(パソコン)にLinuxをインストールするのが一番おすすめ. ただ、余っているPCなど常人は持っていないので、WindowsユーザならWSL2を使うことを推奨する.
責任とれないので勧めはしないが、デュアルブートもあり. RaspberrypiはLinux入るので慣れるのには良いかも.
macOSユーザはBSD系で多少コマンドやシステム構成がLinuxと違うが、基本同じなので学べることは多い(BSDもUnix派生). Linux, macOSなどのことをよくUnix-likeなOSなどと呼ぶ.

仮想環境(VitualBox)などは重いのでおすすめしない. 普段使いには向いていないが開発時にDockerを使うのはおすすめ.

## What is Linux

1991年に`Linux Torvalds`がC言語で書き始め、公開したカーネル. 本来`Linux`とはカーネルの名前であり、OSの名前ではない. 
`Windows`でいうところのNTカーネルや、`macOS`でいうところのXNUにあたる. しかし、一般的にはLinux DistributionsのことをLinuxと呼ぶ.

## Distribution(Distro)とは

ディストリビューション、またディストロとはLinux環境をまとめて配るパッケージのことである. 簡単に言えば、Windowsなどの一般的なOSのことである.
というのも、カーネルだけでは何もすることはできないからである. 車で言うエンジンのようなもの.

Distroには沢山の種類があり、よく話題に上がるのは以下

- Ubuntu
  一番有名. 大体皆これ使ってる. PC雑魚いと意外と重い. 割と安定してる.
  とりまこれ使えたらなんとかなる.

- LinuxMint
  デスクトップでLinux使ってる人たちの中ではかなり有名. UbuntuとはGUI違うが親戚みたいなもん.

- Debian
  UbuntuやLinuxMintの先祖(派生distroをdebian-basedってよく言う). 安定. サーバ用途でよく見る.

- CentOS
  サポート切れたがサーバで有名. RedHat系ってやつ(RedHatは企業向けLinuxみたいな感じ. IBM傘下).

- ArchLinux
  KISS(Keep It Simple, Stupid)を地で行くOS. yanoのメインOS. 自分でやることが多く初心者には若干厳しめ.

- Manjaro
  Arch-basedだが初心者向け.

- raspberrypiOS
  ラズパイでよく使うOS. debian-based. 割と軽い.

他にも勿論星の数程あるので(言い過ぎ)、distrowatch.comなんか見てみると面白いかもしれない.
初心者はとりあえずUbuntu使っとけば間違いない. ロボット制御で使われるのも十中八九Ubuntu.
